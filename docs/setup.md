# 🚀 Настройка и установка AI Smart Grabber

## 📋 Требования

### Оборудование
- **ESP32-CAM** (AI-THINKER модель)
- **ESP32** DevKit
- **Ультразвуковые датчики** HC-SR04 (3 шт.)
- **Сервоприводы** SG90 или MG90S (4 шт.)
- **Манипулятор/роботическая рука**
- **Блок питания** 5V 2A
- **Соединительные провода**

### Программное обеспечение
- **Arduino IDE** 1.8.19+
- **Python** 3.8+
- **Git**

## 🔧 Настройка окружения

### 1. Установка Arduino IDE и библиотек

## 📱 Настройка ESP32-CAM

### 1. Подключение ESP32-CAM

```
ESP32-CAM ↔ Программатор FTDI
VCC ↔ 5V
GND ↔ GND
U0R ↔ TX
U0T ↔ RX
IO0 ↔ GND (для прошивки)
```

### 2. Конфигурация Wi-Fi

Откройте `esp32_cam/config.h` и измените:

```cpp
#define WIFI_SSID "YOUR_WIFI_SSID"
#define WIFI_PASSWORD "YOUR_WIFI_PASSWORD"
```

### 3. Прошивка

1. Подключите IO0 к GND
2. Подключите питание
3. В Arduino IDE выберите плату: "AI Thinker ESP32-CAM"
4. Выберите правильный COM порт
5. Загрузите скетч `esp32_cam/camera_web_server.ino`
6. Отключите IO0 от GND и перезагрузите

### 4. Проверка

Откройте Serial Monitor (115200 baud). После подключения к Wi-Fi вы увидите IP адрес. Откройте его в браузере.

## 🦾 Настройка ESP32-Servo

### 1. Подключение компонентов

#### Ультразвуковые датчики:
```
Левый датчик:
TRIG → GPIO5
ECHO → GPIO18
VCC → 5V
GND → GND

Центральный датчик:
TRIG → GPIO19
ECHO → GPIO21
VCC → 5V
GND → GND

Правый датчик:
TRIG → GPIO22
ECHO → GPIO23
VCC → 5V
GND → GND
```

#### Сервоприводы:
```
Серво основания → GPIO12
Серво плеча → GPIO13
Серво локтя → GPIO14
Серво захвата → GPIO15
VCC → 5V
GND → GND
```

### 2. Конфигурация

Откройте `esp32_servo/config.h` и настройте:

```cpp
// Wi-Fi настройки
#define WIFI_SSID "YOUR_WIFI_SSID"
#define WIFI_PASSWORD "YOUR_WIFI_PASSWORD"

// IP адрес AI сервера
#define AI_SERVER_HOST "192.168.1.10"  // IP вашего ноутбука

// IP адрес ESP32-CAM
#define ESP32_CAM_URL "http://192.168.1.100/capture"
```

### 3. Прошивка

1. Подключите ESP32 к компьютеру
2. Выберите плату: "ESP32 Dev Module"
3. Загрузите скетч `esp32_servo/manipulator.ino`

## 🤖 Настройка AI сервера

### 1. Конфигурация

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
ESP32_CAM_URL=http://192.168.1.100/capture
MODEL_PATH=yolov8n.pt
CONF_THRESHOLD=0.5
```

### 2. Запуск сервера

```bash
cd ai_server
python main.py
```

Сервер будет доступен по адресу: `http://localhost:8000`

### 3. Проверка API

```bash
# Проверка статуса
curl http://localhost:8000/status

# Проверка разрешенных классов
curl http://localhost:8000/classes

# Анализ изображения с камеры
curl -X POST http://localhost:8000/analyze_from_camera
```

- **ESP32-Servo**: 192.168.1.101


## 🧪 Тестирование системы

### 1. Тестирование ESP32-CAM

1. Откройте IP адрес камеры в браузере
2. Проверьте захват изображения: `http://IP_АДРЕС/capture`
3. Проверьте статус: `http://IP_АДРЕС/status`

### 2. Тестирование AI сервера

1. Откройте `http://localhost:8000/docs` для Swagger документации
2. Протестируйте эндпоинт `/analyze_from_camera`
3. Проверьте детекцию на тестовых изображениях

### 3. Тестирование полной системы

1. Разместите объект перед манипулятором
2. Наблюдайте за логами в Serial Monitor ESP32-Servo
3. Система должна:
   - Обнаружить объект (расстояние < 15см)
   - Запросить анализ у AI сервера
   - Выполнить захват если объект разрешен

## 🔧 Поиск неисправностей

### Проблема: ESP32-CAM не подключается к Wi-Fi
- Проверьте SSID и пароль в config.h
- Убедитесь что Wi-Fi сеть 2.4GHz
- Перезагрузите устройство

### Проблема: AI сервер не отвечает
- Проверьте что сервер запущен: `python main.py`
- Проверьте IP адрес и порт
- Проверьте файрвол на ноутбуке

### Проблема: Манипулятор не двигается
- Проверьте подключение сервоприводов
- Убедитесь что питание 5V достаточно
- Проверьте пины в config.h

### Проблема: YOLO модель не загружается
- Убедитесь что файл модели существует
- Проверьте права доступа к файлу
- Попробуйте скачать модель вручную:
  ```python
  from ultralytics import YOLO
  YOLO('yolov8n.pt')
  ```
